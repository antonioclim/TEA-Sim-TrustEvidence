# FHIR R4/BALP-facing mapping note

The TrustEvidence envelope is a project-defined evidence object. Selected fields are conceptually mapped to FHIR R4 AuditEvent, Provenance and Consent categories and to BALP privacy-audit patterns. The mapping helps reviewers understand semantic correspondence; it is not a serialisation profile and does not establish FHIR or BALP conformance.

No SUSHI, IG Publisher or HL7 FHIR Validator run is part of the v2.1.0 core execution. Historical v2.0.1 work recorded SUSHI compilation with warnings while IG Publisher and the HL7 FHIR Validator CLI were unavailable. Those historical logs are not promoted as v2.1 validation evidence.
