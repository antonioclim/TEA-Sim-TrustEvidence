# Retained CMPB v2.1.0 reference evidence

This directory contains the reviewed reference outputs for the integrated `v2.1.0` package. The result files are separated by reproducibility class rather than treated as uniformly byte-invariant.

## Deterministic evidence

Schema decisions, required-field deletions, competency-question statuses, canonicalisation vectors, controlled mutation decisions, property-test summaries and finite bounded-check summaries are regenerated by the corresponding public scripts. Their retained CSV/JSON outputs are compared byte-for-byte where the protocol declares them byte-deterministic.

## Workload evidence

The workload reference passage uses deterministic synthetic monitoring-accountability events at 128, 512 and 2,048 leaves with 12 repetitions per size. Event identities, counts, receipt/proof sizes and acceptance or rejection decisions are reproducible under the declared protocol. Wall-clock append and verification measurements are single-host observations and are not expected to be byte-identical across computers.

Raw event and timing rows are retained under `raw_runs/`. The public package contains no raw physiological observations, direct participant identifiers or payload-commitment nonces.

## Result contracts and provenance

Top-level result CSVs are checked against Draft 2020-12 contracts in `schemas/results/` by `scripts/validate_result_contracts.py`. `reproducibility_manifest.csv` records the SHA-256 value, evidence class and generator or source for every retained result file except itself, thereby avoiding circularity.

## Interpretation boundary

Passing these local checks supports reproducibility of the reference implementation and the stated finite experiments. It is not a formal proof, clinical validation, production benchmark, standards-conformance result, legal-compliance assessment or guarantee of global non-equivocation.
