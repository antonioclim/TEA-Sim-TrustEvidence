# FHIR validation instructions

Static repository checks are available through `make validate-fsh-static` and `make validate-ig-static`. Full FHIR Shorthand compilation requires SUSHI and access to the FHIR R4 package cache. Full implementation-guide publication requires the HL7 IG Publisher JAR.

The repository contains draft artefacts only. Successful local static checks do not constitute FHIR, BALP or HL7 conformance.
