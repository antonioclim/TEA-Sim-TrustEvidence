# Limitations and claim boundary

This kit supports the following wording only after the bundled logs are inspected:

> Local reference-implementation execution was completed for the A2 Merkle backend, including deterministic canonicalisation, append receipts, inclusion proof verification, prefix consistency checks and tamper-rejection tests under the declared runtime.

It does **not** support the following claims:

- PostgreSQL performance was measured in this runtime;
- Rekor, Trillian or Hyperledger Fabric executed locally;
- the system is production-ready;
- the system is clinically validated;
- the package establishes legal compliance;
- the Merkle implementation is a cryptographic security proof;
- the benchmark is a production blockchain or database benchmark.
