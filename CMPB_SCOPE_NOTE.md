# CMPB scope note

This release is organised as a biomedical computing methods package. Its unit of analysis is a synthetic personal-monitoring accountability event, and its method curates audit evidence about how monitoring objects are registered, accessed, consented, transformed, disclosed, aggregated or rejected.

The package is not a generic governance framework, a ledger comparison, a clinical study or a standards-conformance claim. FHIR R4 and BALP are used only as conceptual mapping sources. The local A2 implementation demonstrates receipt construction and verifier-visible checks under stated assumptions; it does not guarantee global non-equivocation.
